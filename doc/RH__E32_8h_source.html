16 // The buffer in the E32 is 512 bytes, but we arbitrarily limit messages to a maximum of 58 octets
17 // We use some for headers, keeping fewer for RadioHead messages
18 // The E32 sends messages longer than 58 octets in several packets of 58 octets each, and since we dont have any
19 // framing or message level checksums there is a risk that long messages from 2 different sources
20 // could be incorrectly interpreted as a single message

91 /// \class RH_E32 RH_E32.h <RH_E32.h>
92 /// \brief Driver to send and receive unaddressed, unreliable datagrams via a EBYTE E32-TTL-1W
93 /// and similar serial radio transceiver.
94 ///
95 /// Works with
96 /// E32-TTL-1W
97 ///
98 /// Note: it should also be possible to use the E32-TTL-1W with the RadioHead RH_Serial module,
99 /// which will also you to send longer packets, but will require you to use the EBYTE Wireless Module Setting program
100 /// to configure the radio first. In this arrangement the E32 would act as a transparent serial connection.
101 /// This has not been tested by us.
102 ///
103 /// \par Overview
104 ///
105 /// This class provides basic functions for sending and receiving unaddressed,
106 /// unreliable datagrams of arbitrary length to 53 octets per packet.
107 ///
108 /// Manager classes may use this class to implement reliable, addressed datagrams and streams,
109 /// mesh routers, repeaters, translators etc.
110 ///
111 /// Naturally, for any 2 radios to communicate that must be configured to use the same frequency and
112 /// modulation scheme.
113 ///
114 /// This Driver provides an object-oriented interface for sending and receiving data messages with EBYTE
115 /// RFM95/96/97/98(W), Semtech SX1276/77/78/7E32-TTL-1W9 and compatible radio modules. These modules implement
116 /// long range LORA transcivers with a transparent serial interface. With 1W power output the manufacturer
117 /// claims up to 6km range.
118 ///
119 /// This Driver provides functions for sending and receiving messages of up
120 /// to 53 octets on any frequency supported by the radio, in a range of
121 /// data rates and power outputs. Frequency can be set with
122 /// 1MHz precision to any frequency from 410 to 441MHz.
123 ///
124 /// You can use either a hardware or software serial connection.
125 ///
126 /// Tested with Arduino Uno and software serial.
127 ///
128 /// \par Packet Format
129 ///
130 /// All messages sent and received by this Driver conform to this packet format:
131 ///
132 /// - 5 octets HEADER: (LENGTH, TO, FROM, ID, FLAGS)
133 /// - 0 to 53 octets DATA
134 ///
135 /// \par Connecting E32-TTL-1W to Arduino
136 ///
137 /// We tested with Arduino Uno. We used SoftwareSerial on pins 6 and 7) to connect to the E32 module, so
138 /// we could continue to use the only hardware serial port for debugging
139 /// \code
140 /// Arduino E32
141 /// GND----------GND (ground in)
142 /// 5V-----------VCC (5V in)
143 /// pin D4-----------M0 (mode control pin input to radio)
144 /// pin D5-----------M1 (mode control pin input to radio)
145 /// pin D6-----------RXD (serial data input from Arduino to radio)
146 /// pin D7-----------TXD (serial data output from radio to Arduino)
147 /// pin D8-----------AUX (Aux pin output from radio to Arduino)
148 /// \endcode
149 /// With this connection, you can initialise the serial port and RH_E32 like this:
150 /// \code
151 /// SoftwareSerial mySerial(7, 6);
152 /// RH_E32 driver(&mySerial, 4, 5, 8);
153 /// \endcode
154 ///
155 /// For Adafruit M0 Feather:
156 /// \code
157 /// Feather E32
158 /// GND----------GND (ground in)
159 /// 3V-----------VCC (3.3V in)
160 /// pin D5-----------M0 (mode control pin input to radio)
161 /// pin D6-----------M1 (mode control pin input to radio)
162 /// pin D1/Tx--------RXD (serial data input from M0 to radio)
163 /// pin D0/Rx--------TXD (serial data output from radio to M0)
164 /// pin D9-----------AUX (Aux pin output from radio to M0)
165 /// \endcode
166 /// With this connection, you can initialise serial port 1 and RH_E32 like this:
167 /// \code
168 /// RH_E32 driver(&Serial1, 5, 6, 9);
169 /// \endcode
170 /// Other connection schems are possible provided the approporiate constructors are used for SoftwareSerial and RH_E32
171 ///
172 /// \par Memory
173 ///
174 /// The RH_RF95 driver requires non-trivial amounts of memory. The sample
175 /// programs all compile to about 8kbytes each, which will fit in the
176 /// flash proram memory of most Arduinos. However, the RAM requirements are
177 /// more critical. Therefore, you should be vary sparing with RAM use in
178 /// programs that use the RH_E32 driver.
179 ///
180 /// It is often hard to accurately identify when you are hitting RAM limits on Arduino.
181 /// The symptoms can include:
182 /// - Mysterious crashes and restarts
183 /// - Changes in behaviour when seemingly unrelated changes are made (such as adding print() statements)
184 /// - Hanging
185 /// - Output from Serial.print() not appearing
186 ///
187 /// \par Performance
188 ///
189 /// This radio supports a range of different data rates and powers.
190 /// The lowest speeds are the most reliable, however you should note that at 1kbps and with an 13 octet payload,
191 /// the transmission time for one packet approaches 5 seconds. Therefore you should be cautios about trying to
192 /// send too many or too long messages per unit of time, lest you monopolise the airwaves.
193 /// Be a good neighbour and use the lowest power and fastest speed that you can.
194 ///
195 /// Forward Error Correction (FEC) is always enabled in these radios by RH_E32.
196 ///
197 /// \par Range
198 ///
199 /// When running with a power output of 1W and at the slowest speed of 1kbps, this module has an impressive range.
200 /// We tested with:
201 /// E32-TTL-1W (1 W power, 1kbps data rate)
202 /// Single wire antenna with a small meta ground plane at about 1m above ground level
203 /// Arduino Uno
204 /// RadioHead RH_E32 module with e32_client and e32_server sketches
205 /// Packet length 13 octets (total payload 18 octets)
206 /// (and yes, we have an appropriate radio license for that power output)
207 ///
208 /// We were able to get reliable reception over 7km (6 km over ocean and 1 km through low rise residential area)
209 ///
210 /// You can expect less range with lower power outputs and faster speeds.
211 /// You can expect less range in highrise cities.
212 /// You can expect more range with directional antennas.
213 /// You can expect more range with shorter messages.
214 ///
215 /// \par Transmitter Power
216 /// TBA
217 ///
218 /// Caution: the maximum power output of this radio (1W = 30dbM) is almost certainly more than the
219 /// permitted power level for unlicensed users in the ISM bands in most countries. Be sure you comply with your local
220 /// regulations. Be a good neighbour and use the lowest power and fastest speed that you can.
221 ///
